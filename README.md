# 박준영의 책임 원장

토익, 푸시업, 미국 주식 매매 계획을 증빙과 함께 잠그고 실패를 공개 회고로 남기는 개인용 Referee 앱입니다. 기록은 이 컴퓨터 브라우저에 우선 저장되므로 인터넷이 없어도 입력·조회할 수 있습니다.

## 처음 실행하기

1. Node.js 22 이상을 설치합니다.
2. 이 폴더에서 `npm install`을 실행합니다.
3. `npm run dev`를 실행합니다.
4. 화면에 표시된 주소를 브라우저로 열고 로컬 PIN을 만듭니다.

PIN은 서버로 전송되지 않습니다. PIN을 잊으면 앱에서 복구할 수 없습니다.

## 판정 원칙

- 하루는 Asia/Seoul 기준 다음 날 오전 3시에 마감됩니다.
- 토익 주간 기록은 월요일 오전 3시에 마감됩니다.
- 증빙이 없는 기록과 판독 불가능한 기록은 성공에 포함하지 않습니다.
- 모의고사는 모의고사 목표만 1회 증가시킵니다.
- 주식 계획은 제출 즉시 잠기며 수정·삭제할 수 없습니다.
- 계획 외 매매, 주문·체결 내역 누락, 보유 내역 누락, 중복 증빙은 실패 사유입니다.
- 주식 기능은 투자 추천이 아니라 사용자가 세운 규칙의 준수 여부만 기록합니다.

## 비상 루틴 사용법

1. `비상 루틴` 탭에서 루틴 직후 수행할 업무를 한 문장으로 입력합니다. 시작 후에는 바꿀 수 없습니다.
2. 집중 가능성·신체 긴장·정신적 소란을 `상·중·하`로 기록하고 `비상 루틴 시작`을 누릅니다.
3. 20초 타이머를 시작한 뒤 발끝, 손가락 끝, 어깨, 목을 각각 5초씩 수축하고 즉시 이완합니다. 타이머는 일시정지 없이 끝나야 합니다.
4. 입과 코가 크게 보이는 3분 이상의 영상을 선택합니다. 앱은 영상 길이, 분석 가능한 중앙 영역 비율, 반복적인 미세 움직임을 기기 안에서 검사합니다.
5. 영상이 불확실하거나 과거 파일과 중복이면 인정하지 않으므로 새 영상을 촬영합니다. 흔들림과 밝기 변화 자체는 반려 사유가 아닙니다.
6. 시작 후 10분 안에 영상 검증까지 끝내야 합니다. 시간을 넘기면 해당 실행은 실패로 잠깁니다.
7. 검증 후 미리 고정한 업무로 10분 집중 타이머를 시작합니다. 일시정지 없이 끝까지 유지해야 효과 성공으로 인정합니다.
8. 마지막으로 사후 상태를 `상·중·하`로 남깁니다. 상태 변화는 장기 분석 자료이며 성공 여부는 10분 업무 유지로 판정합니다.

여러 번 시작하면 각 실행이 독립된 약속입니다. 시작한 실행을 완료하지 못하면 실패 이력과 다음 날 공개 회고 대상에 포함되며 영상 원본은 공개하지 않습니다.

## 테스트와 빌드

- `npm test`: 판정 규칙 테스트와 배포 빌드를 실행합니다.
- `npm run build`: 공개 가능한 정적 결과물을 만듭니다.

## GitHub Pages 공개

저장소의 Settings → Pages에서 Source를 **GitHub Actions**로 선택합니다. main 브랜치에 변경이 올라가면 앱이 자동 게시됩니다. 실패 회고 원문과 로컬 기록은 사용자가 명시적으로 게시하기 전에는 저장소로 전송되지 않습니다.

## AI 증빙 검토

일반 기록은 파일 해시 중복 탐지와 보수적인 수동 확정 흐름을 제공합니다. 비상 루틴 영상은 외부 서버로 보내지 않고 브라우저에서 길이와 중앙 입·코 영역의 반복 움직임을 분석합니다. 이는 의료적 호흡량이나 ADHD 상태를 진단하는 기능이 아닙니다. 조명, 압축, 촬영 각도에 따라 불확실 판정이 날 수 있으며 불확실하면 성공으로 세지 않고 재촬영합니다.

## 보안과 개인정보

- API 키나 GitHub 토큰을 코드, 브라우저 입력란, 공개 저장소에 넣지 마세요.
- 공개 회고에는 증빙 사진, 종목, 수량, 손익, 계정 정보가 포함되지 않아야 합니다.
- 브라우저 데이터를 지우면 로컬 기록도 사라질 수 있으므로 정기 백업 기능이 향후 필요합니다.
- 앱의 잠금은 자기 통제를 돕지만 컴퓨터와 GitHub 계정 소유자의 우회를 기술적으로 완전히 막지는 못합니다.

- 로컬 실패 회고를 GitHub에 안전하게 자동 전송하는 중계 서비스
